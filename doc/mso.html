1 Introduction

MSOamp is a computer code to evaluate the elastic scattering observables for nucleon-nucleus elastic scatering from an optical potential in the momentum space configuration. The optical potential is constructed as a multiple scattering expansion in terms of the free NN transition amplitude and the target densities[1,2,3]. The free NN transition amplitude is calculated in the momentum space configuration from a realistic NN interaction such as Paris or Bonn [4,5]. This is an ideal tool to study the isospin dependence of the optical potential and thus to to perform comparative studies of neutron and proton elastic scattering in the intermediate energy region. It can be used to support fundamental studies planned in n-TOF.

The code is constructed in fortran 90. The inputs are given through a user friendly namelist.

Details of the formalism and some applications can be found in [6].

3 About the code

3.1 Source files

3.2 Source files and compilation

The source files of the MSOamp program are organized into two directories, nnsrc and msosrc. The former contains the NNamp source files, while the latter contains those files specific of the MSO formalism. Each one of these directories contains a Makefile. There exits also a Makefile in the top directory of the distribution. If needed, this can be edited and customized before compilation. Notice that the compiler name and options are included by means of *.def files. Presently, two of these files are included: fujitsu.def and pgf90.def. They contain the compilation specifications for the Fujitsu and PGF90 compilers, respectively. If you have a different f90 compiler, you may want to create your own *.def file, with the appropriate options.

3.3 Compilation

Once the top Makefile have been customized, the NNamp program can be compiled with: 

prompt> cd nnsrc ; make

Similarly, to compile the MSOamp program: 

prompt> cd msosrc; make

Alternatively, from the top directory to can compile the NNamp program typing: 

prompt> make nnamp

and the MSOamp program with: 

prompt> make msoamp

Notice that, whenever the MSO program is compiled, the NNamp program will also be compiled.

After compilation, you can delete the *.o files by typing: 

prompt> make clean

To delete *.o as well as MSOamp / NNamp output files: 

prompt> make superclean

To install the executables in the compilation (mso and nnamp) into the destination specified at the top Makefile, type: 

prompt> make install

4 Description of namelists input variables

MSO namelist:

Genearal options:

  • tlab: relative kinetic energy projectile-target in laboratory system.
  • offshell: T/F
  • kmt: KMT (kmt=T) or Watson (kmt=F) potential. The former contains the multiplying factor at/(at-1), with at the target atomic number.
  • igwf: If T, calculates wavefunction up to lstore.
  • lstore: max. partial wave for wf.
  • nr: if nr is not 0, a relativistic correction is performed.
  • ucnr, ucni: central renormalization constants.
  • usnr, usni: s.o. renormalization constants.

COUL namelist:

  • coulomb: Coulomb correction
    =0: no coulomb
    =1: coulomb amplitude and phase change included 
    =2: coulomb amplitude, but not phase change
    =3: realistic (HO shape) charge distribuion
    =3: uniform charge sphere distribution
  • rcut :cut-off radius for subtracted momentum space method [8].
  • alfa, beta, gama: parameters for the realistic charge distribution of the form:
    (24)

HIGHORDER namelist:

Specifications for high order calculations (second order onward)

  • second:
    = 1 vopt1
    = 2 vopt1 + vopt2(local)
    = 3 vopt1 + vopt2 (non local)
    = 4 vopt1 + vopt2 (full nonlocal)
  • isflip:not used
  • ich: not used
  • ion2: not used
  • lamx:not used
  • lbmx: not used
  • l1mx:not used
  • lfmx:not used
  • rmaxr: maximum radius
  • quin: maximum radius for inner region
  • mquadi: number of inner quad points (multiple of 6)
  • mquado: number of outer quad points (multiple of 6)

PROJ namelist:

Projectile properties. Presently, the program only supports structureless projectiles. Thus, unlike the target, no clusters can be defined.

  • massp: mass number (in a.m.u.)
  • zp: charge

TARG namelist:

Target properties

  • masst: mass number (in a.m.u.)
  • zt: charge
  • at: number of nucleons
  • ncluster: number of clusters

CLUSTER namelist:

After the targ namelist, ncluster namelists will be read. Each one contains the information required to construct the density for one the clusters. Notice that the different cluster do not neccesary have to use the same model density.

  • type, shape: specify the kind of density. External densities (type=1, shape=5) are read from fortran unit 4. The first line contains the number of points (nramax) and the step (drx), in free format. Then, nramax density points will be read, assuming that the first one corresponds to the origin.
    type shape
    0 Numerical potential 0 Woods-Saxon
        1 Gauss
        2 Yukawa
        3 Hulthen
        4 cosh
        5 External
    1 Analytical potential 0 Harmonic-oscillator s.p.
        1 Three-parameter Fermi
        2 G3 distribution
  • nzclus: number of protons within this cluster.
  • nnclus: number of neutrons within this cluster.

BSPARM namelist:

For type=0, shape=0-4 the bsparm namelist is then read.

  • cmass, vmass: core and bound particle masses.
  • zc, zv: core and bound particle charges.
  • nramax: number of integration steps.
  • drx: radial step for numerical integration.
  • dmat:fine tuning of interior-exterior matching. Usually input 0. * If no state found try increase of dmat to of order 10.
  • nshell: number of shells.

BSHELL namelist:

Parameters for this specific shell.

  • bengy: binding energy.
  • vdepth, wr0, wal: potential depth, radius and diffuseness.
  • wls: spin-orbit strength.
  • is2=2*valence particle spin .
  • lmoma= orbital angular momentum .
  • j2a : 2*j, where .
  • nodd=number of nodes.
  • norba=occupation number of this shell.

  • w3p, z3p, c3p: Fermi parameters for protons.

For type=1, shape=0 densities, harmonic oscillator parameters are read. First, a hoparm namelist is introduced, which currently only contains the number of shells that will be next read by means of hoshell namelists.

HOSHELL namelist:

Provides the HO parameters and occupations numbers for each shell.

  • aap, aan: HO parameters for protons and neutrons, respectively.
  • zorba, norba: occupation number (protons and neutrons) for this shell.

5 Test examples

5.1 p+O with HO formactors

In describing O we consider a shell model harmonic oscillator (HO) description where the center-of-mass corrections are neglected. Thus, the nucleus matter density distribution is directly evaluated from the HO single particle wave functions using the relative frequency obtained by Donnelly and Walker [9]. 


# MSO input file: mso_o16_ho.in 
# p + 16O scattering at E=50 MeV including Coulomb
  &mso tlab=500 offshell=T kmt=T  lxmax=35 
       ngp=40, kode=022 b=21000. nang=-1 ymin1=-7. ymin2=-2. nr=0 
       ucnr=1. ucni=1. usnr=1. usni=1. /
  &coul coulomb=3 rcut=10.  alfa=0.3916125 beta=3.1329 gama=0. /
  &highorder second=1 closure=F rmaxr=10. quin=6. mquadi=120 mquado=48 /
  &proj massp=1.0072 zp=1 /
  &targ ncluster=1 masst=16. at=16 zt=8 /
  &cluster type=1 shape=0 nzclus=8 nnclus=8 /

# HO
  &hoparm   nshell=2  /
  &hoshell  aap=1.77  aan=1.77 zorba=2 norba=2 /
  &hoshell  aap=1.77  aan=1.77 zorba=6 norba=6 /

5.3 p+Li with external (Hartree-Fock) formfactors

In describing the Li ground state, we use a shell model Hartree-Fock description as described in the work of [11]. This is read externally from fortran unit file 4. The first line of this line contains the number of points and radial step. 


# MSOamp input file: mso_li9_hf.in
# p + 9Li scattering at E= MeV including Coulomb
# Reads external form factor from unit 4
  &mso tlab=100 offshell=T kmt=T  lxmax=35 
       ngp=40, kode=022 b=21000. nang=-1 ymin1=-7. ymin2=-2. nr=0 
       ucnr=1. ucni=1. usnr=1. usni=1. /
  &coul coulomb=4 rcut=10.  alfa=0.3916125 beta=3.1329 gama=0. /
  &highorder second=1 closure=F rmaxr=10. quin=6. mquadi=120 mquado=48 /
  &proj massp=1.0072 zp=1 /
  &targ ncluster=1 masst=9. at=10 zt=3 /
  &cluster type=0 shape=5  /

5.4 p+Li with mixed (cluster) formfactors

As a check of of the dependence of the scattering observables on the model adopted for the matter density, we repeat in this example the calculation for p+Li, with a different model for the Li. Now we assume a cluster model for Li as in [12] consisting of an -particle-like core, four neutrons in the shell coupled to spin zero, and a proton in the shell. We take a Gaussian distribution for the nucleons in the alpha core. For the other nucleons we consider two cases: an harmonic oscilator model (here called Cluster I) where the valence neutrons and protons are described within the harmonic oscillator single particle model. The ranges of the parameters correspond to the parameter set (a) found in [12]. We also take the valence nucleons as bound to the core by a Woods-Saxon potential with parameters adjusted to obtain the valence neutron and proton separation energy (here called cluster II). 


# MSO input file: mso_li9_clus.in
# Reaction: p + 9Li @ 100 MeV
# Cluster form factor: alpha + 4n + 1p
  &mso tlab=100. offshell=T kmt=T ioptls=1 ioptcnt=0 lxmax=25 
       ngp=40, kode=022 b=21000. nang=-1 ymin1=-7. ymin2=-2. nr=0 
       ucnr=1. ucni=1. usnr=1. usni=1. /
  &coul coulomb=4 rcut=10.  alfa=0.3916125 beta=3.1329 gama=0. /
  &highorder second=1 closure=F rmaxr=10. quin=6. mquadi=120 mquado=48 /
  &proj massp=1.0072 zp=1 /
  &targ ncluster=3 masst=9. at=9 zt=3 /
 
# alpha cluster
  &cluster type=1 shape=0  /
  &hoparm   nshell=1  /
  &hoshell  aap=1.38  aan=1.38 zorba=2 norba=2 /

# 4n
  &cluster type=0 shape=0  /
  &bsparm  cmass=8  vmass=1 zc=3 zv=0.
          nramax=212  drx=0.1  dmat=0. nshell=1  /
  &bshell  ia=0  bengy=4.1  vdepth=0  
         wr0=1.37  wal=0.61  wls=0. 
         j2a=1 lmoma=1 is2=1  nodd=0  norba=4   /
  &bsparm / 
 
# 1p
  &cluster type=0 shape=0  /
  &bsparm  cmass=8  vmass=1 zc=2 zv=1
          nramax=212  drx=0.1  dmat=0. nshell=1  /
  &bshell  ia=0  bengy=13.9  vdepth=0  
         wr0=1.37  wal=0.61  wls=0. 
         j2a=1 lmoma=1 is2=1  nodd=0  norba=1   /

  &bsparm /

#Uses Paris potential

      s=1.0 qlo=0.  qhi=100. /

     itype=2 icase=2 /
